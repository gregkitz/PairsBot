# Developer Requests

This directory contains requests for the developer that require external resources, infrastructure changes, or additional permissions. The requests are organized by category.

## Current Requests

- **CI/CD Integration**: [ci_cd_integration.md](./ci_cd_integration.md) - Request for setting up GitLab CI/CD pipeline for automated testing and deployment.

## How to Use This Directory

When the development team needs something from the project owner or requires external resources, we'll add a new file in this directory with the following structure:

1. A clear title describing the request
2. Detailed explanation of what is needed
3. Justification or benefits for the project
4. Any timeline considerations
5. Contact information for follow-up

## Request Status

Requests will be tagged with one of the following statuses:

- `[NEW]` - Newly created request awaiting review
- `[PENDING]` - Request is under consideration
- `[APPROVED]` - Request has been approved and is being implemented
- `[COMPLETED]` - Request has been fulfilled
- `[REJECTED]` - Request has been rejected (with explanation)

## Adding New Requests

When adding new requests, please create a new Markdown file with a descriptive name following the pattern `request_name.md` and follow the structure of existing requests.
